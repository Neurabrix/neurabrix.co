ClinicOS doctor portal

Doctor Dashboard

The dashboard gives a clinician one place to open the live queue, scan patient readiness, and start the first patient action without jumping between schedule, notes, and roster screens.

Live queue Booked and walk-in patients stay visible in the first screen.
Patient context Risk, registration, token, timing, and diagnosis readiness are shown together.
First action The doctor can move from queue review into notes or patient detail.
Demo video: dashboard BVT capture with synthetic doctor queue data. Upgrade target: CreateDemoV2 clarity-master

Workflow shown

1

Open the doctor dashboard

The portal lands on the operational dashboard with the clinician identity and today queue already loaded.

2

Review the queue table

The doctor sees position, patient name, registration number, time, token, and diagnosis state in one compact table.

3

Confirm the active patient context

The first queued patient remains stable across the dashboard so the clinician can start from the correct record.

4

Move into patient work

The dashboard exposes the action path into notes and patient detail without making the doctor search again.
